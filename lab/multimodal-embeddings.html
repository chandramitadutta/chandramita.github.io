Multimodal Embeddings

What happens when you embed images and text in the same space? Playing with CLIP to build a personal semantic search over my photo library.

The Question

CLIP (Contrastive Language–Image Pre-Training) learns a shared embedding space for images and text — meaning you can compare them directly with cosine similarity. A photo of a dog and the text "a golden retriever in a park" end up close together in vector space.

The practical question I wanted to answer: can I search my own photo library with natural language? Not "DSC_4821.jpg" but "that foggy morning in the city" or "the photo where the light was interesting."

The Approach

embed_photos.py — encode all photos with CLIP and store vectors 
import torch
import clip
from PIL import Image
from pathlib import Path
import numpy as np
import faiss

device = "cuda" if torch.cuda.is_available() else "cpu"
model, preprocess = clip.load("ViT-B/32", device=device)

def embed_photos(photo_dir: str) -> tuple[np.ndarray, list[str]]:
    paths, embeddings = [], []
    for img_path in Path(photo_dir).rglob("*.jpg"):
        try:
            image = preprocess(Image.open(img_path)).unsqueeze(0).to(device)
            with torch.no_grad():
                emb = model.encode_image(image)
                emb /= emb.norm(dim=-1, keepdim=True)  # normalize
            embeddings.append(emb.cpu().numpy())
            paths.append(str(img_path))
        except Exception:
            continue
    return np.vstack(embeddings).astype("float32"), paths

def search(query: str, index: faiss.Index, paths: list[str], k: int = 5):
    with torch.no_grad():
        text = clip.tokenize([query]).to(device)
        text_emb = model.encode_text(text)
        text_emb /= text_emb.norm(dim=-1, keepdim=True)
    query_vec = text_emb.cpu().numpy().astype("float32")
    distances, indices = index.search(query_vec, k)
    return [(paths[i], float(distances[0][j])) for j, i in enumerate(indices[0])]

What's Working, What Isn't

The semantic search works surprisingly well for scene-level queries: "sunset over water", "crowded street", "close-up of food" all return reasonable results. It's less reliable for emotional or atmospheric queries — "a quiet moment", "something melancholy" — which is interesting in itself.

Observation: CLIP's shared embedding space encodes visual and textual semantics, but not quite the same semantics a human uses when they remember a photo. I remember photos by how they felt; CLIP knows them by what's in them. That gap is the interesting part.
