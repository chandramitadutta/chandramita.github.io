Overview

I've been using Obsidian for note-taking for about two years. The problem with any note-taking system is that notes accumulate faster than connections — you end up with a lot of individual thoughts but no map of how they relate to each other.

This project is an attempt to fix that. I parse my Obsidian vault, extract entities and relationships, load them into a Neo4j graph database, and then use Cypher queries to explore the structure. The goal isn't to build a search engine — I have that. The goal is to find clusters, bridges, and unexpected paths between ideas I hadn't consciously connected.

Architecture

The pipeline has three stages: parse, extract, load.

parse_vault.py — walk the Obsidian vault and extract raw note data 
import os
import re
from pathlib import Path
from dataclasses import dataclass, field
from typing import List

@dataclass
class Note:
    title: str
    path: str
    body: str
    tags: List[str] = field(default_factory=list)
    links: List[str] = field(default_factory=list)  # [[wikilinks]]

def parse_vault(vault_path: str) -> List[Note]:
    notes = []
    for md_file in Path(vault_path).rglob("*.md"):
        body = md_file.read_text(encoding="utf-8")
        # Extract [[wikilinks]]
        links = re.findall(r'\[\[([^\]]+)\]\]', body)
        # Extract #tags
        tags  = re.findall(r'#(\w[\w/-]*)', body)
        notes.append(Note(
            title=md_file.stem,
            path=str(md_file),
            body=body,
            tags=list(set(tags)),
            links=[l.split('|')[0].strip() for l in links],
        ))
    return notes

The extraction stage uses a lightweight NER pass to pull out named entities (people, concepts, places) and classify them as nodes. Links between notes become edges. Tags become node labels.

load_graph.py — push nodes and relationships into Neo4j 
from neo4j import GraphDatabase

class KnowledgeGraph:
    def __init__(self, uri: str, user: str, password: str):
        self.driver = GraphDatabase.driver(uri, auth=(user, password))

    def add_note(self, note: Note):
        with self.driver.session() as session:
            session.run(
                """
                MERGE (n:Note {title: $title})
                SET n.path = $path, n.tags = $tags
                """,
                title=note.title, path=note.path, tags=note.tags
            )
            for linked_title in note.links:
                session.run(
                    """
                    MERGE (a:Note {title: $from_title})
                    MERGE (b:Note {title: $to_title})
                    MERGE (a)-[:LINKS_TO]->(b)
                    """,
                    from_title=note.title, to_title=linked_title
                )

Querying the Graph

The interesting part is what Cypher lets you ask. Standard Obsidian search is keyword-based — this lets you ask structural questions.

Find notes that bridge two topic clusters 
# Cypher: find high-betweenness nodes (manual approximation)
query = """
MATCH (n:Note)
WHERE size((n)--()) > 10
WITH n, size((n)-->()) AS out_degree, size((n)<--()) AS in_degree
RETURN n.title AS note,
       out_degree + in_degree AS total_connections
ORDER BY total_connections DESC
LIMIT 20
"""
Unexpected finding: The most-connected bridging note in my vault isn't about data science at all — it's a note about urban planning and how cities encode the assumptions of their designers. It connects to a cluster about algorithmic bias, a cluster about infrastructure, and somehow also to a cluster about pedagogy. I hadn't noticed that connection consciously.

What I'm Still Figuring Out

The pipeline works, but the graph quality depends heavily on how consistently I've written notes. Notes with lots of wikilinks are well-connected; older notes I wrote in isolation are orphans. I'm thinking about whether to retroactively add links or to accept the graph as a faithful record of how I was thinking at the time.

The bigger open question: what's the right way to visualize this? Neo4j Browser gives me a force-directed graph but it becomes unreadable past ~200 nodes. I'm exploring whether D3.js or a custom Python visualization would let me highlight specific subgraphs — show me everything in the "data ethics" cluster and its first-degree neighbors, for instance.

Next step: experiment with graph embeddings (Node2Vec) to see if I can find semantically similar notes that aren't explicitly linked.
